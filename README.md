
# Retail Data Warehouse - ETL решение для ритейл-аналитики

Проект многослойного хранилища данных для анализа продаж, клиентов и товарных остатков в ритейле. Полный цикл ETL от исходных данных до готовых аналитических витрин.

## Технологии и архитектура

- **Docker-контейнеризация** для изолированного и воспроизводимого развертывания всей инфраструктуры
- **Многослойная архитектура DWH**: Staging → ODS → CDM → Marts
- **dbt (data build tool)** для трансформации данных и управления моделями
- **PostgreSQL** как основная база данных
- **SQL** для ETL-процессов
- **DBeaver** для подключения и работы с данными

## Что реализовано

- Полный ETL-конвейер от сырых данных до аналитических витрин
- 4 слоя данных с четким разделением ответственности
- dbt-модели с документацией и тестами качества данных
- Готовые аналитические витрины для отделов продаж и маркетинга
- Возможность подключения через DBeaver для прямого доступа к данным

## Сильные стороны проекта

**Контейнеризация** - весь стек развертывается в Docker одной командой, без зависимостей и конфликтов

**Производственная готовность** - архитектура соответствует стандартам enterprise-уровня

**Масштабируемость** - легко добавлять новые источники данных и метрики

**Прозрачность данных** - полная трассировка от витрин до исходных систем

**Гибкость** - возможность адаптации под конкретные бизнес-задачи

## Как подключиться к проекту

### Через DBeaver:
1. Установите DBeaver
2. Создайте подключение к PostgreSQL
3. Используйте параметры из конфигурации проекта
4. Все таблицы и витрины доступны для анализа
